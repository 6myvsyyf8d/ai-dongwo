## 一、产品定位

### 1.1 综合定位

本产品是面向心智障碍者（心青年）的**综合支持平台**，三位一体：

- **动态支持档案**：为心青年建立动态、可视化的个人支持档案
- **多方协作平台**：支持家长、老师、护理员、志愿者、心青年本人多方协同记录和查看
- **智能决策系统**：基于AI能力提供照护支持决策，聚焦情绪行为问题

### 1.2 核心价值

**信息共享与沟通**——打破多方之间的信息断层，让照护信息在家长、老师、护理员、志愿者之间高效流转。

### 1.3 社会价值

系统不仅服务个体照护，还承载数据积累的社会价值链条：

```
多角色日常记录 → 结构化档案沉淀 → 个体照护支持（即时价值）
                  → 脱敏数据聚合 → 机构/区域统计分析（管理价值）
                  → 科研数据输出 → 政策制定参考（社会价值）
```

### 1.4 核心痛点

| 痛点 | 说明 |
|------|------|
| 信息孤岛 | 家长、老师、护理员之间信息不互通 |
| 信息碎片化 | 心青年信息散落各处，没有统一记录 |
| 交接成本高 | 新接手的老师/护理员需要很长时间熟悉心青年 |
| 应急响应难 | 情绪波动时缺乏快速参考和应对指导 |

## 二、信息架构

### 2.1 三层信息结构

```
第一层：速览层（Dashboard）—— 按角色定制的首页，展示最关键的信息和待办
第二层：档案层（Profile）—— 结构化的心青年完整档案，按主题分类
第三层：记录层（Records）—— 日常记录入口，支持多种记录类型
```

### 2.2 档案层主题分类

| 主题 | 包含内容 | 对应现有功能 |
|------|---------|-------------|
| 基本档案 | 姓名、年龄、性格特点、沟通方式 | basicInfo |
| 喜好档案 | 喜欢/不喜欢的事物、活动偏好 | renderLife |
| 沟通档案 | 沟通指南、有效话术、禁忌用语 | renderCommunication |
| 情绪档案 | 情绪触发因素、安抚策略、历史趋势 | renderEmotion + 情绪记录 |
| 照护档案 | 日常照护要点、注意事项、作息 | renderCare |
| 支持档案 | 工作能力、社交关系、支持网络 | renderWork + renderRelation |

### 2.3 角色定制视图

每个角色登录后看到的首页内容不同：

- **家长**：最新记录、情绪趋势、待办任务、其他角色反馈、AI建议
- **老师**：沟通指南、学习表现、情绪状态、速读卡入口
- **护理员**：照护要点、作息安排、情绪预警、应急方案
- **志愿者**：速读卡、基本沟通方式、活动建议、注意事项
- **心青年本人**：今日任务、心情记录、简单导航

### 3.1 角色视图矩阵

```
志愿者   ●●○○○       ●●○○○       ●●○○○       ●●○○○
心青年   ●●●○○       ●●○○○       ●●●○○       ●●○○○
```

### 3.2 各角色详细权限

**家长**
- 速览：最新记录、情绪趋势、待办任务、其他角色反馈、AI建议
- 档案：全部档案（除本人隐私外）
- 记录：可录入所有类型，可查看所有记录
- AI：全量建议、预警、策略推荐、效果分析
- 特殊权限：管理用户、设置隐私级别、管理紧急联系人

**老师**
- 速览：沟通指南、学习表现、情绪状态、今日任务
- 档案：沟通/情绪/喜好/工作能力档案（不含家庭隐私）
- 记录：可录入学习/行为/沟通记录，可查看相关记录
- AI：沟通建议、情绪预警、行为策略推荐
- 特殊权限：导出班级报告（脱敏）

**护理员**
- 速览：照护要点、作息安排、情绪预警、应急方案
- 档案：照护/情绪/喜好档案（不含教育/工作详情）
- 记录：可录入照护/情绪/行为记录，可查看相关记录
- AI：照护建议、情绪预警、行为策略推荐
- 特殊权限：查看当班期间所有记录

**志愿者**
- 速览：速读卡入口、基本沟通方式、注意事项、活动建议
- 档案：基本档案 + 喜好档案 + 沟通档案（仅基础信息）
- 记录：可录入活动/沟通记录，查看权限有限
- AI：沟通建议、活动推荐
- 特殊权限：仅活动期间有访问权限

**心青年本人**
- 速览：今日任务、心情记录入口、简单导航
- 档案：自己的基本档案（简化版）
- 记录：可录入心情/笔记
- AI：简单的情绪识别和自我调节建议
- 特殊权限：自己的数据自己拥有

### 3.3 隐私分级体系

| 级别 | 说明 | 可见角色 | 数据示例 |
|------|------|---------|---------|
| A级（公开） | 基本信息和喜好 | 所有角色 | 姓名、年龄、喜欢的事物 |
| B级（照护） | 日常照护信息 | 家长、老师、护理员 | 照护要点、作息、沟通方式 |
| C级（敏感） | 行为和情绪详情 | 家长、指定护理员 | 情绪记录、行为问题、策略使用 |
| D级（私密） | 家庭和医疗信息 | 仅家长 | 家庭关系详情、医疗诊断、用药信息 |
| E级（科研） | 脱敏后数据 | 经授权的科研/机构 | 聚合统计数据（无身份信息） |

### 3.4 数据权限控制

```javascript
function canAccess(userRole, privacyLevel, recordType) {
  const permissions = {
    parent:    { A: true, B: true, C: true, D: true, E: false },
    teacher:   { A: true, B: true, C: 'partial', D: false, E: false },
    caregiver: { A: true, B: true, C: 'partial', D: false, E: false },
    volunteer: { A: true, B: false, C: false, D: false, E: false },
    self:      { A: true, B: 'own', C: 'own', D: false, E: false }
  };
  return permissions[userRole]?.[privacyLevel] || false;
}
```

### 3.5 数据价值层原则

| 原则 | 说明 |
|------|------|
| 标准化采集 | 记录字段采用标准化分类，便于后续聚合分析 |
| 分级授权 | 个体数据、机构数据、科研数据三级访问权限 |
| 脱敏处理 | 数据输出时自动脱敏，保护心青年隐私 |
| 可导出 | 支持按机构/区域导出结构化数据（CSV/JSON） |

## 四、AI能力层 - 聚焦情绪行为支持

### 4.1 核心聚焦

孤独症群体最大的问题是情绪和行为问题，AI能力聚焦于**情绪行为问题解决策略推荐**。

### 4.2 AI能力架构

```
┌──────────────────────────────────────────────────┐
│           情绪行为支持引擎（核心）                  │
├──────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐  │
│  │ 情绪识别  │→│ 原因分析  │→│ 策略推荐       │  │
│  │ 与预警    │  │ 与模式发现│  │ 与效果跟踪    │  │
│  └──────────┘  └──────────┘  └───────────────┘  │
├──────────────────────────────────────────────────┤
│  数据基础                                          │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌──────────┐  │
│  │情绪记录 │ │行为记录 │ │触发情境 │ │策略知识库 │  │
│  └────────┘ └────────┘ └────────┘ └──────────┘  │
└──────────────────────────────────────────────────┘
```

### 4.3 四个核心AI能力

#### 4.3.1 情绪行为识别与预警
- 实时识别：根据近期情绪/行为记录判断当前状态
- 趋势预警：发现情绪恶化趋势，提前预警
- 模式发现：识别周期性情绪规律（如每周三情绪波动）
- 输出示例：⚠️ "近3天情绪记录显示焦虑指数上升，建议关注"

#### 4.3.2 触发原因分析
- 情境关联：分析情绪/行为问题与特定情境的关联
- 历史对比：对比类似情境下的历史表现
- 多因素分析：综合考虑作息、天气、活动、社交等因素
- 输出示例：📊 "情绪波动多发生在午休后，可能与感官过载有关"

#### 4.3.3 解决策略推荐（核心能力）
- 基于情绪类型推荐策略：焦虑→深呼吸/安静空间；暴躁→运动释放
- 基于个人档案匹配：根据心青年喜欢的安抚方式个性化推荐
- 分级策略：轻度→自我调节；中度→照护者介入；重度→专业支持
- 策略知识库结构：情绪类型 → 严重程度 → 推荐策略 → 注意事项

#### 4.3.4 策略效果跟踪与优化
- 记录每次策略使用及效果
- AI分析哪些策略对个人最有效
- 持续优化推荐，形成个人化策略库
- 输出示例：✅ "上次焦虑时使用'安静空间+音乐'策略有效，建议优先使用"

### 4.4 闭环反馈机制

```
情绪/行为问题发生
      │
      ▼
AI识别并推荐策略 ──→ 用户记录策略使用
      │                    │
      ▼                    ▼
策略效果评估 ←── AI分析有效性
      │
      ▼
优化个人策略库（越用越准）
```

### 4.5 策略效果评估数据结构

```javascript
{
  strategyUsed: "深呼吸引导",
  emotionType: "焦虑",
  severity: "轻度",
  trigger: "考试前",
  effectiveness: 4,           // 效果评分 1-5
  timeToEffect: "8分钟",
  notes: "配合感官玩具效果更好"
}
```

### 4.6 技术实现方式

| 层面 | 方案 |
|------|------|
| 规则引擎 | 前端JS实现预设规则，处理确定性场景 |
| LLM调用 | 后端API代理调用（隐藏API Key），前端通过接口获取结果 |
| 知识库 | 结构化JSON存储照护专业知识，可扩展 |
| 数据隐私 | 调用LLM前进行脱敏处理，不传输身份信息 |

## 五、策略知识库

### 5.1 知识库结构（三级展开）

```
一级：情绪/行为类别
  └── 二级：严重程度 / 情境
       └── 三级：具体策略卡
            ├── 策略名称
            ├── 适用情境
            ├── 实施步骤（分步）
            ├── 注意事项
            ├── 禁忌
            ├── 预期效果
            └── 效果评估方法
```

### 5.2 覆盖的情绪/行为类型

#### 原始六类
1. 焦虑/紧张（轻/中/重）
2. 暴躁/攻击行为（轻/中/重）
3. 刻板/重复行为
4. 社交退缩/拒绝
5. 睡眠问题
6. 饮食问题

#### 新增四类
7. 自伤行为（轻/中/重）
8. 恐惧/恐怖反应（情境性恐惧）
9. 强迫/重复行为
10. 多动/冲动行为

### 5.3 特定情境策略

| 情境 | 覆盖阶段 |
|------|---------|
| 就医 | 就医前、候诊、检查、就医后 |
| 理发 | 理发前、理发中、替代方案 |
| 公共交通 | 乘车前、乘车中、突发情况 |
| 超市购物 | 购物前、购物中、结账 |
| 新环境适应 | 进入前、初次进入、逐步适应、出现问题 |

### 5.4 年龄差异化策略

| 年龄段 | 特点 | 策略调整方向 |
|--------|------|-------------|
| 儿童（6-12岁） | 语言能力有限、依赖照护者 | 大量视觉支持、家长主导、游戏化 |
| 青少年（13-17岁） | 自我意识增强、同伴压力 | 自我调节、社交应对、独立性培养 |
| 成人（18岁+） | 多重角色、长期模式 | 就业支持、自主决策、社会融入 |

### 5.5 感官支持策略

#### 感官饮食（Sensory Diet）
- 本体觉：推墙、深蹲、搬重物、深压力按摩
- 前庭觉：秋千、转椅、跳跃、平衡木
- 触觉：触觉板、沙盘、橡皮泥
- 视觉：调暗灯光、暖色调、减少杂乱
- 听觉：白噪音、降噪耳机、自然声音
- 嗅觉：薰衣草、柑橘等精油（偏好测试后）

#### 感官友好环境设计
- 家中：柔和灯光、安静角落、感官工具箱
- 教室：靠近老师、减少装饰、使用隔板
- 公共场所：感官工具包（降噪耳机/墨镜/安抚物）
- 工作场所：安静工位、弹性休息

#### 感官过载应对流程
1. 识别预警信号（捂耳/闭眼/烦躁/退缩）
2. 立即降低环境刺激
3. 提供深压力输入
4. 允许自我调节
5. 恢复后逐步重新参与
6. 记录触发因素和有效策略

> 注：策略知识库详细内容（含所有策略卡的完整实施步骤）将在实现阶段以结构化JSON格式录入，本文档记录分类框架和覆盖范围。

## 八、成功标准

### 8.1 阶段一成功标准

1. 不同角色登录后看到不同的首页内容
2. 档案信息按新主题分类组织，查找效率提升
3. 隐私分级控制生效，不同角色看到的字段不同
4. 记录系统交互优化，录入更便捷
5. 现有数据兼容，不丢失历史记录

### 8.2 整体成功标准
