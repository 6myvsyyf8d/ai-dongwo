# 阶段一实现计划 - 信息架构重构

> 基于设计文档：2026-07-21-requirements-redesign-design.md  
> 日期：2026-07-21

## 改动范围

仅修改 `js/app.js` 和 `index.html`，保持纯前端架构。

## 实现步骤

### 步骤1：重构隐私分级系统

**改动文件**：js/app.js

**当前状态**：4级隐私（A=最私密，D=最公开），方向与设计文档相反

**目标状态**：5级隐私（A=公开，D=私密，E=科研）

**改动点**：
1. `privacyLevels` 配置（~L213）：改为5级体系
2. `applyPrivacy` 函数（~L3149）：适配新分级逻辑
3. `renderCare` 中的 `data-privacy` 属性（~L2373）：更新标记值
4. `renderRelations` 中添加隐私标记（~L2486）：家庭关系标记为D级

**新隐私级别映射**：
| 级别 | 说明 | 可见角色 |
|------|------|---------|
| A | 公开（基本信息） | 所有角色 |
| B | 照护（日常照护） | 家长、老师、护理员 |
| C | 敏感（情绪行为） | 家长、护理员 |
| D | 私密（医疗家庭） | 仅家长 |
| E | 科研（脱敏数据） | 特殊授权 |

### 步骤2：实现角色定制首页

**改动文件**：js/app.js

**改动点**：
1. `renderHome()`（~L1594）：根据角色渲染不同的导航卡片
2. `renderWelcomeBanner()`（~L1671）：根据角色显示不同的引导信息
3. `renderLatestActivity()`（~L1699）：根据角色过滤记录类型

**各角色导航卡片**：
- 家长：完整档案、动态时间轴、每日任务、数据可视化
- 老师：沟通说明书、速读卡、每日任务、日程日历
- 护理员：照护要点、情绪预警、日程日历、速读卡
- 志愿者：速读卡、沟通方式、今日活动、注意事项
- 心青年：记录心情、今日任务、日程日历、我的档案

### 步骤3：添加档案统一入口

**改动文件**：index.html, js/app.js

**改动点**：
1. index.html：新增"档案"页面section
2. app.js：新增 `renderArchive()` 函数，展示六大主题分类入口
3. routeMap：添加 archive 路由

### 步骤4：测试验证

1. 分别用5个角色登录验证首页内容差异
2. 验证隐私控制：志愿者看不到医疗信息
3. 验证档案入口功能
4. 确保现有数据兼容

## 成功标准

1. 不同角色登录后看到不同的首页内容
2. 隐私分级控制生效，不同角色看到的字段不同
3. 档案分类入口可用
4. 现有数据兼容，不丢失历史记录
